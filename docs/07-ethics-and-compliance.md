# 07. 合规和使用边界

爬虫项目必须先讲边界。

本仓库是抖音评论采集项目的方法论复盘和交付模板，不提供可直接滥用的大规模采集配置，不包含真实账号、cookie、浏览器 profile、验证码处理凭证或真实客户数据。

## 推荐原则

- 只处理公开可见数据。
- 遵守目标平台服务条款、robots、频率限制和当地法律。
- 不采集敏感个人信息。
- 不绕过强认证、付费墙或访问控制。
- 不发布真实用户评论原文，除非已获得授权或完成充分脱敏。
- 不把低量数据伪造成完整数据。
- 不用无关评论凑数。
- 不在开源仓库提交账号、cookie、token、密钥、`.env`。

## 数据脱敏建议

公开展示时，建议保留结构，不保留真实内容：

| 原始字段 | 开源样例处理 |
|---|---|
| 用户昵称 | 替换为 `user_001` |
| 用户 ID | 删除或哈希 |
| 评论正文 | 改写为英文/中文虚构样例 |
| 视频 URL | 替换为 `https://example.com/video/...` |
| 客户目标清单 | 替换为 `Sample Model A` |
| 采集时间 | 保留日期格式即可 |

## 频率和稳定性

真实项目中，不建议为了追求速度盲目提高并发。

更稳的做法：

- 先跑样例。
- 再跑小批量。
- 观察异常率。
- 遇到验证或异常状态时暂停。
- 记录失败原因。
- 对低量目标做说明。

## 适用范围

适合：

- 公开评论研究。
- 数据采集流程设计。
- 外包交付模板。
- 作品集案例复盘。
- 数据质量校验学习。

不适合：

- 绕过平台限制。
- 大规模滥采。
- 账号池、验证码绕过、黑灰产用途。
- 收集或传播个人敏感信息。
