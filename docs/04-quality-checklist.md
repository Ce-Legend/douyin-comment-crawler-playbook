# 04. 质量校验清单

采集结果必须能被检查。下面是评论类数据交付的最小校验标准。

## 结构校验

- 顶层是否是列表。
- 每个目标是否有目标名。
- 每个目标是否有视频列表。
- 每个视频是否有视频 ID。
- 每个视频是否有视频 URL。
- 每个视频是否有评论列表。
- 每条评论是否有评论 ID。
- 每条评论是否有评论内容。
- 每条评论是否有评论时间。

## 数量校验

- 总目标数。
- 非 0 目标数。
- 0 数据目标数。
- 达到目标量的目标数。
- 每个目标有效评论数。
- 每个目标视频数。
- 每个视频评论数。

## 时间校验

- 评论时间是否存在。
- 评论时间是否可解析。
- 评论时间是否在需求范围内。
- 同一视频下评论排序是否异常。

## 去重校验

优先使用评论 ID 去重。

如果没有稳定评论 ID，可以组合：

```text
目标名 + 视频 ID + 评论内容 + 评论时间
```

交付报告中要说明去重键。

## 有效性校验

自动脚本只能做一部分判断，文本有效性最好结合抽样。

建议抽样检查：

- 每个高量目标抽 5 条。
- 每个低量目标抽 3 条。
- 每个别名搜索目标抽 3 条。
- 每个新增策略目标抽 3 条。

抽样时看：

- 是否讨论目标本身。
- 是否明显广告。
- 是否只有表情或无意义文本。
- 是否可能归错目标。

## 交付前最终清单

```text
[ ] 完整 JSON 可打开
[ ] 按目标拆分文件可打开
[ ] 视频 URL 字段存在
[ ] 评论时间字段存在
[ ] 时间范围无越界
[ ] 重复评论数为 0 或已解释
[ ] 低量目标已列出
[ ] 字段说明已提供
[ ] 质量报告已提供
[ ] 不包含账号、cookie、profile、密钥、客户原始资料
```
